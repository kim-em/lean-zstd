## What was accomplished

Completed Track C3, the final FSE fuel removal in `Zstd/Native/Fse.lean`:

- Split the fuel-driven `for _ in [:fuel]` loop in `decodeFseSymbolsAll` into
  an inner recursive helper `decodeFseSymbolsAllLoop` and a slim
  outer wrapper. The outer `decodeFseSymbolsAll` no longer takes a
  `fuel` parameter (issue verification requirement met).
- `decodeFseSymbolsAllLoop` is structurally recursive on a `Nat` safety
  budget — no explicit `termination_by`/`decreasing_by` needed. The
  wrapper seeds the budget as
  `br.totalBitsRemaining + 2 * (1 <<< table.accuracyLog)`, i.e. "bits
  remaining in the backward stream, plus a table-size slack for chains
  of 0-bit cells". This is the "or equivalent" variant of the natural
  bit-reader measure called out by the plan.

## Why structural safety instead of a pure bit measure

An earlier draft used `br.totalBitsRemaining` directly with a runtime
guard `hadv : br''.totalBitsRemaining < br.totalBitsRemaining`. That
compiled cleanly but broke the `Huffman integration` native test
(Test 86): real Huffman FSE tables contain cells with `numBits = 0`,
so the 2-state interleave can emit several symbols without consuming
bits. A pure bit-based measure terminates a round too early and
returns a truncated weight array, which downstream
`buildZstdHuffmanTable` rejects with "all weights are zero".

A lexicographic `(totalBitsRemaining, safety)` measure would work but
requires a `readBits_totalBitsRemaining_le` lemma that currently only
lives in `Zstd.Spec.Fse.BackwardBitReader`; the Native module can't
depend on Spec. Rather than duplicate the lemma into Native (or
refactor the import order), I used structural recursion on the safety
budget. The budget is *derived from* `totalBitsRemaining` at the call
site, so the termination argument is still tied to the bit stream.

## Spec side

`parseHuffmanWeightsFse_succeeds` and `parseHuffmanWeightsFse_pos_eq`
in `Zstd/Spec/Huffman.lean` use `decodeFseSymbolsAll` only through its
signature (`= .ok (weights, bbr')`). Removing the default `fuel`
parameter does not change the signature of calls without a fuel
argument, and both `parseHuffmanWeightsFse` and
`ZstdTest/NativeIntegration.lean` were already calling without fuel.
No spec changes were needed.
