## Issue
#25 — Feature: Split Spec/Content.lean into thematic submodules

## Accomplished
Split `Zstd/Spec/Content.lean` (1292 lines) into four thematic submodules
under `Zstd/Spec/Content/`, mirroring the merged Huffman (#21) and
Sequence (#23) split patterns:

- `Single.lean` (124 lines) — single-block frame content (compressed
  literals-only, compressed sequences)
- `CompressedFirst.lean` (573 lines) — frames with a compressed first
  block (5 frame theorems + 1 block-loop helper + compressed_seq+rle
  content)
- `SimpleFirst.lean` (445 lines) — frames with a raw or RLE first block
  (4 frame theorems + 2 block-loop helper lemmas)
- `TwoCompressed.lean` (241 lines) — homogeneous two-compressed frames
  (two literals-only, two sequences)

The top-level `Content.lean` is now a 12-line re-export shim, so
downstream imports (`Zstd/Spec/Complete.lean`, `Zstd.lean`) work
unchanged.

## Decisions
- Grouped theorems by the type of the first block (single / compressed
  first / raw-rle first / two-compressed homogeneous) rather than by
  the existing `/-! ## -/` banners, because the banners were sparse
  (only 2 in the file) and the first-block-type partition yields more
  balanced submodule sizes.
- Duplicated the `frame_from_blocks` local macro into each submodule
  that contains a frame theorem (Single, CompressedFirst, SimpleFirst,
  TwoCompressed — all four). The `unfold_except` macro declared at the
  top of Content.lean was dropped because it was never used.
- CompressedFirst.lean slightly exceeds the ~500-line practical target
  (573 lines) but stays well under the 800-line hard limit. A further
  split by second-block type felt artificial because the theorems share
  common compressed-first proof structure.
