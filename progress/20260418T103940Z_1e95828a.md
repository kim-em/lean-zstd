## Decisions / patterns discovered

The `decodeFseLoop.induct` principle produces 8 cases (one per branch of the
match tree in the function body). Slot counts per case were determined
empirically via `trace_state` — some slots correspond to let-bindings
(nameable) and one slot is auto-reserved for the last hypothesis before the
motive (must be grabbed via `rename_i`).

Hypothesis types often carry let-bindings (e.g. `hp0 : (_prob == 0) = true`
where `_prob := Int32.ofNat val - 1`). To bridge these to the simp-target
form after `rw [decodeFseLoop.eq_def]`, we use
`have hp0 : (Int32.ofNat val - 1 == 0) = true := hp0` to zeta-expand the let.
For goals, `change T at h` / `change T` does the same work for propositional
positions.

The `case7` (non-zero, non-(-1), non-overflow) branch uses `change` to
translate between `_prob.toInt` and `int32ToNat _prob` forms.
