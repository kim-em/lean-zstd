# Split Spec/Sequence.lean into thematic submodules (issue #14)

**Date**: 2026-04-18 UTC
**Session**: feature / c41ad47e
**Issue**: #14

## What was accomplished

Split `Zstd/Spec/Sequence.lean` (1701 lines) into 8 thematic submodules under
`Zstd/Spec/Sequence/`, following the pattern established by the recent
Huffman.lean and Fse.lean splits. The original file is replaced by a slim
re-export shim.

All files well under the 800-line target. Largest is `Validity.lean` at 370.

## Key decisions

- **Moved `executeSequences_output_length`** from the original ExtraBits section
  to `Execute.lean` — it's about execution output, not extra bits. Semantic
  placement over textual order.
- **Two namespaces preserved**: `Zstd.Native` (for theorems about native
  definitions) and `Zstd.Spec.Sequence` (for spec-level predicates and their
  consequences). Matches the structure of the source file.
- **No cross-file private theorems**: verified each `private` helper is used
  only within its target file before splitting.
- **`Validity.lean` independent of `Primitives.lean`**: the validity layer
  uses only native definitions (`resolveOffset`, `executeSequences.loop`);
  `Execute.lean` is the module that depends on both.
